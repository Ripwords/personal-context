# Braindump Plan C — Document RAG

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** Upload documents, store + chunk + index their text, and retrieve relevant chunks for the chat — reliably and self-hosted.

**Decision (overnight reliability):** Files are stored on the **local filesystem** (`./uploads`, gitignored; a Docker volume in compose) and chunk retrieval uses **Postgres full-text search** (same proven pattern as memory). Text is extracted for `.txt`/`.md` (UTF-8) and `.pdf` (via `unpdf`, a pure-JS extractor). **Upgrade path (documented, not built):** swap storage to `rustfs` (S3) and retrieval to pgvector+Ollama embeddings — `searchDocuments`/the store interface change, nothing else.

**Architecture:** `document` + `document_chunk` (with generated `tsvector`+GIN) tables. An upload route saves the file, extracts text, chunks it, inserts chunks. `searchDocuments(db, query)` FTS-ranks chunks and returns them with their document. A documents page manages uploads; `searchDocuments` is the backend for the future `search_documents` chat tool.

**Tech Stack:** Drizzle/node-postgres (+FTS), `unpdf` (PDF text), Nuxt 4 Nitro multipart, Nuxt UI, Bun.

## Task 1: tables + chunking + FTS search

**Files:** `server/db/schema.ts` (+migration), `server/rag/chunk.ts` (+test), `server/db/queries/documents.ts` (+test).

**Interfaces:**
- `document`: `id uuid pk`, `filename text`, `mimeType text`, `sizeBytes integer`, `storagePath text`, `createdAt timestamptz default now()`.
- `document_chunk`: `id uuid pk`, `documentId uuid → document.id (cascade)`, `chunkIndex integer`, `content text`, `createdAt`. Hand-edited migration adds `search tsvector GENERATED ALWAYS AS (to_tsvector('english', content)) STORED` + GIN index (same approach as Plan D `memory`). Update `truncateAll` to include `document_chunk, document`.
- `chunkText(text: string, opts?: { maxChars?: number }): string[]` (pure) — splits on paragraph/sentence boundaries into ~2000-char chunks; drops empty. TDD directly.
- `createDocumentWithChunks(db, doc, chunks): Promise<{ documentId: string; chunks: number }>` (transaction: insert document + its chunks).
- `listDocuments(db)`, `deleteDocument(db, id)` (cascade deletes chunks).
- `searchDocuments(db, query, limit=10): Promise<Array<{ documentId: string; filename: string; chunkIndex: number; content: string }>>` — FTS over `document_chunk.search` (`websearch_to_tsquery`, ts_rank), joined to `document` for filename; empty query → most recent chunks.

## Task 2: upload + management routes + text extraction

**Files:** `server/rag/extract-text.ts` (+test), `server/api/documents/index.get.ts`, `server/api/documents/index.post.ts`, `server/api/documents/[id].delete.ts`.

**Interfaces:**
- `extractText(filename: string, mimeType: string, bytes: Uint8Array): Promise<string>` — `.txt`/`.md`/`text/*` → UTF-8 decode; `.pdf`/`application/pdf` → `unpdf` `extractText`; otherwise throw `Error("unsupported file type")`. (`bun add unpdf`.)
- `POST /api/documents` (multipart, 401-gated): read the uploaded file (h3 `readMultipartFormData`), validate size (≤ ~10MB) + type, write bytes to `./uploads/<uuid>-<safe-filename>`, `extractText`, `chunkText`, `createDocumentWithChunks`; return `{ documentId, filename, chunks }`. On unsupported type → 400.
- `GET /api/documents` → `listDocuments`. `DELETE /api/documents/[id]` → delete row (cascade) + unlink the stored file (ignore missing file).
